# Wardrobe Wizard Architecture

Wardrobe Wizard is a Python + Streamlit Creative Apps project with a local rule-based recommendation engine, GitHub Models integration when configured, a TypeScript Rewear Impact mini API deployed to Azure Container Apps, and standalone MCP tools for agent-ready workflows.

The project is intentionally not a fully autonomous agent. It is an AI wardrobe assistant with controlled, inspectable logic and human review around model-generated outputs.

## Runtime architecture

```text
User
→ Streamlit Cloud
→ app.py
→ Streamlit Session State + data/wardrobe.json
→ local recommendation_engine.py
→ GitHub Models for explanation/photo analysis when configured
→ local fallbacks when external AI is unavailable
→ impact_client.py
→ Azure Container Apps Rewear Impact mini API when configured
→ local Python Rewear Impact fallback when API is unavailable
```

## Main components

| Component | Role |
|---|---|
| `app.py` | Streamlit UI, navigation, session state, human review tables |
| `data/wardrobe.json` | 36 synthetic sample wardrobe items |
| `src/recommendation_engine.py` | Deterministic outfit scoring and outfit assembly |
| `src/item_parser.py` | Natural-language item parsing for user-added wardrobe items |
| `src/ai_client.py` | GitHub Models explanation path and local explanation fallback |
| `src/photo_analyzer.py` | Clothing-only vision analysis path and safe fallback |
| `src/impact_client.py` | Rewear Impact API client and local Python fallback |
| `impact-api/` | TypeScript + Express mini API for Rewear Impact estimates |
| `mcp_server/` | Standalone MCP tools for gap detection, safety review, and outfit debugging |

## Recommendation flow

1. The user chooses a scenario, up to two style vibes, and a comfort preference.
2. For Travel day, the user may add departure climate, destination climate, season, and flight type.
3. The app reads the current session wardrobe.
4. `recommendation_engine.py` scores items against scenario, style tags, weather tags, formality, comfort, and favorites.
5. The highest-scoring items become the Main Outfit.
6. The Alternative Outfit is generated by rerunning the same scoring while excluding Main Outfit items.
7. `ai_client.py` explains the recommendation if GitHub Models is configured; otherwise a local explanation is used.

No generative model chooses the outfit. The model can help explain the result, but the selection logic remains local and inspectable.

## Photo analysis flow

1. The user uploads a JPG or PNG.
2. `photo_analyzer.py` sends a clothing-only prompt to a vision-capable model when a token is configured.
3. The prompt explicitly forbids identity recognition, face analysis, body comments, attractiveness judgments, and sensitive inferences.
4. Detected clothing items are shown in an editable review table.
5. Only reviewed items are added to the session wardrobe.
6. If the model is unavailable, the app returns a safe fallback message and the rest of the app still works.

## Rewear Impact flow

1. The recommended outfit is sent to `src/impact_client.py`.
2. If `IMPACT_API_URL` is configured, the client calls the TypeScript mini API.
3. If the API URL is missing or unavailable, the app uses local Python estimates.
4. The UI shows which calculation mode was used.

The impact estimate is educational. It compares rewearing current wardrobe items with producing similar new items. It is not a certified environmental calculator.

## Azure Container Apps

The mini API is designed to run as a small containerized service:

```text
impact-api/
→ TypeScript
→ Express
→ Docker multi-stage build
→ Azure Container Apps
→ HTTPS endpoint
```

The intended cost-control configuration is:

```text
min replicas = 0
max replicas = 1
```

## MCP tools

The MCP server is not part of the Streamlit runtime path. It exposes selected wardrobe logic as standalone tools:

- `closet_gap_detector`
- `privacy_safety_auditor`
- `outfit_debug_report`

This makes the project agent-ready without pretending that the Streamlit app is a fully autonomous agent.

## Foundry IQ status

A Foundry IQ / Azure AI Search proof-of-concept was documented earlier. The search service was intentionally deleted for cost control. Foundry IQ is not an active production dependency of the current Streamlit app.

The `knowledge/` folder remains useful as project knowledge for a future Foundry recreation or agent extension.

## GitHub Copilot

GitHub Copilot is represented as an AI-assisted development tool. It helped during the development workflow, but it is not a runtime backend and does not serve model responses to users.

See the visual diagram in `docs/architecture_diagram.md`.
